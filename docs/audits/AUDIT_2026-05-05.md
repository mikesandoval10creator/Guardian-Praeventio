## 1. Sumario ejecutivo

Guardian Praeventio es un producto **arquitectónicamente serio** — 23 routers Express, ~40 servicios, 218 archivos de test, mutation testing al 72.37 % global y diseño multi-tenant pensado. La calidad del andamiaje supera el promedio de proyectos comparables.

Sin embargo, entre la promesa documentada (`STATE_OF_FUNCTIONALITY_2026-05-04.md` declara ~99 % E2E) y el comportamiento real del código existen **brechas materiales** que un lanzamiento comercial expondría de inmediato:

1. **El SOS no llega a los teléfonos de la brigada.** El backend lee `fcmToken` (singular) en `projects/{id}/members/{uid}`, mientras el endpoint que registra tokens los escribe en `users/{uid}.fcmTokens` (plural, array). No hay sincronización entre paths. El push siempre devuelve `notified: 0`. (`src/server/routes/emergency.ts:98-99` vs `src/server/routes/push.ts:56`).
2. **La detección de caída no escala.** `FallDetectionMonitor.tsx:87` contiene literalmente `// Here we could also trigger a real emergency protocol via EmergencyContext`. Solo emite una notificación local; al expirar el countdown no se dispara protocolo real.
3. **Las reglas de Firestore no cubren `tenants/*`.** El servidor escribe a `tenants/{tenantId}/...` por Admin SDK, pero `grep tenants firestore.rules` retorna vacío. Cualquier escritura cliente (offline, PWA) es denegada por default-deny sin warning explícito.
4. **El IAP de Apple no concede beneficios.** `validate-receipt` solo hashea y devuelve 202; Apple no tiene webhook real (solo Google Play tiene RTDN). iOS jamás activa el tier pagado.
5. **Vertex AI nunca se selecciona.** `vertexAdapter.ts:50` declara `readonly isAvailable = false` hardcoded. La promesa de residencia de datos en `southamerica-west1` cae silenciosamente al consumer Gemini en `us-central1`.

Adicionalmente: `POST /api/maintenance/check-overdue` y `GET /api/environment/forecast` están sin `verifyAuth`; el i18n Latam (es-MX, es-AR, es-PE) tiene ~10 % de claves; los webhooks billing no emiten audit-log; y el módulo CPHS / DIAT no existe más allá de strings de UI.

**Recomendación**: no lanzar comercialmente hasta cerrar los 8 hallazgos P0 (≈29 SP). Los 11 hallazgos P1 (≈57 SP) son aceptables como deuda gestionada con plan público. La base técnica permite remediar en ~6 sprints de un dev senior.

## 4. Hallazgos por dominio

### 4.1 IA + Modo Crisis

#### Vertex AI residencia de datos (H4)

```ts
// src/services/ai/vertexAdapter.ts:42-50
class VertexAdapter implements AiAdapter {
  readonly name: AiProvider = 'vertex-ai';
  readonly region: string;
  /** Hard-coded `false` until Round 2 installs `@google-cloud/aiplatform` */
  readonly isAvailable = false;
```

**Impacto**: clientes Latam que paguen por residencia regional (`southamerica-west1`) reciben respuestas desde `us-central1` sin saberlo. Desde Ley 19.628 (Chile) y Ley 25.326 (Argentina) puede ser non-compliance contractual.

**Remediación**: instalar `@google-cloud/aiplatform`, completar el adapter, conmutar el flag, agregar test que verifique selección por `tenant.dataResidency`.

#### FallDetection no escala (H6)

```tsx
// src/components/emergency/FallDetectionMonitor.tsx (≈ línea 87)
addNotification({ title: 'Posible Caída Detectada', ... });
setShowModal(false);
// Here we could also trigger a real emergency protocol via EmergencyContext
```

`handleNeedHelp` también dice `// Trigger real emergency protocol here`. La feature comercial **no existe end-to-end**.

**Remediación**: invocar `useEmergency().triggerSOS({ kind: 'fall', autoNotify: true })`. Añadir test que mockee el countdown a 0 y verifique despacho.

#### Push de brigada vacío (H7)

`emergency.ts:98-99` lee `data?.fcmToken` (singular) en `projects/{id}/members/{uid}`. `push.ts:56` registra `users/{uid}.fcmTokens` (plural, `arrayUnion`). **Nadie sincroniza**.

**Remediación opciones**:

1. Cambiar `emergency.ts` para leer `users/{memberDoc}.fcmTokens` (cross-collection lookup) y enviar a todos los tokens del array.
2. O bien al registrar el token (`push.ts`), copiar al doc de miembro de cada proyecto activo del usuario.

La opción 1 es preferible (single source of truth) pero más costosa por reads. Recomendar opción 1 con cache (TTL 5min) o pre-fetch batch.

#### Geofence con bypass latente (H11)

`useGeofence.ts:179` tiene `// eslint-disable-next-line react-hooks/exhaustive-deps`. El effect que registra `watchPosition` solo re-corre cuando el id de zona cambia. Editar la geometría in-place (mismo id, distintos vértices) deja al watcher con la geometría anterior. **Vector de bypass de geofence**.

Adicionalmente, error handler de `getCurrentPosition` es `() => {}` silencioso (`useGeofence.ts:170`). Si el navegador deniega permiso, el usuario sigue creyendo que está protegido.

**Remediación**: tracking de `geometryHash` en deps, surface error → toast "Geolocalización denegada — protección desactivada".

#### AsesorChat con geo de Santiago (H12)

```ts
// src/components/AsesorChat.tsx:188-189
const lat = (selectedProject as any)?.coordinates?.lat ?? -33.4489;
const lon = (selectedProject as any)?.coordinates?.lon ?? -70.6693;
```

Faena en Antofagasta o Punta Arenas recibe consultas con clima/normativa de Santiago. Riesgo: respuestas de IA con contexto incorrecto (ej. plan de calor para clima frío).

**Remediación**: si no hay coords, NO pasar contexto geo al prompt; agregar mensaje "Sin coords disponibles — respuestas sin contexto geográfico".

### 4.3 Billing y webhooks

#### IAP Apple sin webhook (H2)

```ts
// src/server/routes/billing.ts:1329-1439
router.post('/iap/apple/validate-receipt', verifyAuth, async (req, res) => {
  const { receiptHash } = ...
  await db.collection('iap_receipt_attempts').doc(...).set({ receiptHash, ... });
  return res.status(202).json({ ok: true });
});
```

Apple Server-to-Server Notifications (SSN) v2 no está implementado. iOS jamás concede beneficios. Pero la UI muestra el botón "Pagar con Apple", recibiendo dinero sin entregar.

**Remediación**: implementar `POST /api/billing/webhook/apple` con verificación de signed payload (JWT con cert chain Apple), wire al mismo `applyEntitlement(uid, sku, tier)` que usa Google RTDN.

Esfuerzo: 13 SP (incluye sandbox de prueba).

#### Webhooks sin audit (H18)

`auditServerEvent` se importa en 6 routers (subscription, onboarding, reports, gamification, telemetry, oauthGoogle), pero NO se llama en webhook handlers de Webpay/MP/Khipu. Replays maliciosos rechazados por idempotencia no quedan trazados.

**Remediación**: añadir `auditServerEvent({ kind: 'billing.webhook.replay', uid, txn, source })` dentro del catch de `withIdempotency`.

## 6. Performance latente

1. **KnowledgeGraph (H22)**: 553 LOC, `force-graph-3d` ~1MB eager, sin worker. Tenant con >1000 nodos congelará main thread. **Acción**: lazy import + Web Worker para layout (`d3-force` worker pattern).
2. **backgroundTriggers (H23)**: bucle síncrono lectura `fcmToken` por miembro tras `get()` paralelo. Lineal con N×500. **Acción**: `Promise.all` con concurrency 10.
3. **App.tsx code-splitting (H24)**: 14 lazy imports en pages, pero componentes pesados eager. **Acción**: `React.lazy` para `KnowledgeGraph`, `Site25DPanel`, `PortableCurriculum`.
4. **Bundle de framer-motion eager**: ~80KB gz. Aceptable, pero candidato a `LazyMotion` con feature splitting.
