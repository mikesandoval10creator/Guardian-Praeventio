## Observed patterns

- The codebase already follows an adapter + facade pattern for error tracking (`getErrorTracker()` in `src/services/observability/index.ts`) with env-based selection and silent degradation. Worth noting because any future product-analytics layer can mirror this shape.
- PII redaction is implemented twice with different rules: browser scrubs GPS lat/lng from breadcrumbs, server scrubs sensitive query-string params. Both strip auth headers.
- Multi-tenant entity model (project / cuadrilla / processes / tasks) exists in the product but no `group()` or tenant-context propagation flows into telemetry.
- The Sentry `setUserContext` bridge is defined and tested but never invoked from runtime code — Sentry only receives `user.id` when a specific `captureException`/`captureMessage` call site explicitly passes `context.userId`.
